# Frequently Asked Questions

Common questions about PanicOnRails - the free, browser-based train track planner.

## General

### What is PanicOnRails?

PanicOnRails is a free, open-source web application for designing model train layouts. It runs entirely in your browser with no installation required, featuring accurate Kato N-Scale track geometry and graph-based train simulation.

### Is PanicOnRails free?

Yes, 100% free. PanicOnRails is open source under the MIT license. No ads, no subscriptions, no hidden costs. Free forever.

### What browsers are supported?

PanicOnRails works in all modern browsers: Chrome, Firefox, Safari, and Edge. We recommend using the latest version for the best experience.

## Usage

### Does it work on mobile?

PanicOnRails is designed desktop-first for precision track placement. It works on tablets, but mobile phones are not recommended due to screen size limitations.

### Can I save my layouts?

Yes! Layouts are automatically saved to your browser's local storage. You can also export layouts as JSON files for backup or sharing.

### Can I share my layouts with others?

Yes, export your layout as a JSON file and share it. Others can import it into their instance of PanicOnRails. URL-based sharing is planned for a future release.

### What track types are available?

Currently focused on Kato N-Scale including:
- Straight tracks (various lengths)
- Curved tracks (various radii)
- Switches/turnouts
- Crossings

More track types are planned for future releases.

## Technical

### Is my data private?

Yes. All data stays in your browser. PanicOnRails does not send your layouts to any server. Your designs are stored locally on your device.

### Can I run PanicOnRails offline?

Currently, you need an internet connection to load the app. PWA (offline) support is planned for a future release.

### I found a bug. How do I report it?

Please open an issue on [GitHub](https://github.com/chayuto/panic-on-rails/issues) with:
- Steps to reproduce
- Expected vs actual behavior
- Browser and OS information
- Screenshots if helpful

### How can I contribute?

See our [Contributing Guide](../CONTRIBUTING.md). We welcome bug reports, feature suggestions, and pull requests.
